# 阵棋

回合制战棋策略游戏，可在浏览器直接游玩。

核心游戏性是**下棋的感觉**：在格子棋盘上经营走位——聚拢、推击、换位、卡位——再用 AOE 技能一次收割最大收益。数值养成刻意做轻（没有等级和经验值），强度差距来自**装配**与**棋盘上的决策**。

> 架构分层：战斗逻辑为**纯 TypeScript**（`game-core`），不依赖任何引擎；
> 玩家操作（`interaction`）、养成/剧情（`game-meta` + `campaign`）也都是引擎无关的纯逻辑；
> 表现层（**PixiJS 2D**，DOM HUD）只消费 `BattleState` / `BattleEvent` 与各层的 ViewModel，因此可平滑移植到其它引擎或小游戏平台。

## 核心特点

- **走位即策略**：移动与技能顺序自由（先移后攻、先攻后移皆可），移动落点未确认前可撤销重选。敌方行动范围与技能预告全程可见，供你像下棋一样推演。
- **AOE 与位移是主角**：十字/直线/范围等命中形状可旋转；推、拉、聚拢、换位等位移技能用来把敌人「摆」进 AOE 或撞墙、推入陷阱与火焰。
- **所见即所得的预览**：释放前完整预览伤害数字、位移轨迹、落点、撞墙/陷阱触发与可击杀提示，预览与结算跑同一套模拟器。
- **没有等级，只有装配**：技能是**秘卷**、装备与消耗品都是**道具**，统一进背包、在整备界面自由装拆。给谁装什么秘卷 = 决定谁会什么技能；换一套装配就是换一套打法。
- **肉鸽随机掉落**：每关胜利除固定掉落（推进剧情的关键秘卷）外，还按本关**稀有度权重**随机掉落若干装备/消耗品。装备与消耗品分**灰/绿/蓝/红/橙**（普通/优良/稀有/史诗/传说）五档，越往后的关卡越容易掉高稀有度。每周目战利品都不一样，逼你围绕手上这套随机装备重新规划配装与打法。
- **AI 不送死**：敌人 AI 复用同一套模拟器枚举行动并评分，内置扎堆惩罚、直线惩罚、危险地形惩罚，不会主动排成你的 AOE 形状。

## 快速开始

本项目使用 **pnpm** 管理依赖，需要 **Node ≥ 18**（Vite 5 要求；Node 16 会在 `pnpm dev` 时报 `crypto.getRandomValues is not a function`）。仓库已附 `.nvmrc`，可用 `nvm use` 切到匹配版本。

```bash
pnpm install
pnpm dev          # 启动 PixiJS 表现层，打开终端给出的本地地址即可游玩
pnpm test         # vitest 测试（内核 / 交互 / 道具 / 流程 / 数值验证）
pnpm typecheck    # 严格模式类型检查
pnpm build        # 类型检查 + 生产构建
```

> 表现层只保留 **PixiJS**。`game-core`/`interaction`/`game-meta`/`campaign` 为引擎无关纯逻辑，
> 若要新增表现层只需实现 `SessionHost` 与 `CampaignHost`。

## 怎么玩

0. 启动进入**标题页** → 「新游戏」沿剧情战役走：过场 → 战斗 → 掉落结算/整备 → 下一关，共 10 关。顶部下拉为**调试**入口，可直接载入任一关卡。
1. 右侧「行动顺序」显示按**速度初动（CT）**排出的出手序列：每个单位按 `speed` 充能，达阈值即行动，速度越高出手越频繁。轮到的我方单位自动选中并显示**蓝色**可移动范围。
2. 移动与技能**顺序自由**（可先移后攻，也可先攻后移）。移动为**暂定**：未确认前可「↩ 撤销移动」重选落点。
3. 技能菜单不默认弹出（避免遮挡棋盘）——**移动后**或**点击该单位自身**时展开。
4. **黄色**= 施法范围，鼠标移到目标点显示 **橙/红色** AOE 命中、伤害数字、位移箭头、撞墙/陷阱提示。技能必须命中有效目标才能「✓ 释放」（禁止空放）。
5. 敌方回合前有**范围预告**，我方回合可查看**威胁区**，据此规划走位。
6. 单位行动完毕（移动+技能用尽，或点「结束行动」）即自动轮到下一个单位。击败所有敌人获胜。
7. 胜利后进入**结算**：展示本关战利品（固定 + 随机掉落，按稀有度着色），再进**整备**把掉落的秘卷/装备/消耗品装到各单位身上，战斗中也可点「休整」调整装配。

核心循环：**观察站位 → 改造站位（聚拢/推击/换位）→ 释放 AOE → 处理残局**。

## 装配系统

所有可获得之物都是**道具**，进同一个背包，在整备界面装到单位身上：

| 类型 | 说明 |
| --- | --- |
| **秘卷**（技能道具） | 装入单位技能栏，出战时决定该单位可用哪些技能；每张秘卷限定可装备的单位 |
| **装备** | 按槽位（武器/护甲/饰品）提供属性加成（攻击/魔力/防御/生命/速度/移动） |
| **消耗品** | 战斗中使用的一次性道具 |

单位没有等级与经验值，属性 = 基础属性 + 装备加成；关卡奖励发的是道具而不是经验。

**稀有度**：装备与消耗品分五档，稀有度越高属性越强、掉率越低。

| 档位 | gray | green | blue | red | orange |
| --- | --- | --- | --- | --- | --- |
| 名称 | 普通 | 优良 | 稀有 | 史诗 | 传说 |

**掉落构成**：每关奖励 = 固定掉落（`guaranteedDrops`，剧情/教学关键秘卷，必得）+ 随机掉落（`randomDrops`：按本关稀有度权重先抽品质、再从该品质掉落池均匀抽装备/消耗品）。秘卷全部走固定掉落——随机重复秘卷无法装备、只会淤积背包。随机源可注入种子（`Rng`），测试与战役数值验证用固定种子复现，实机每次胜利用新种子。见 [levelRewards.json](src/game-data/levelRewards.json) 与 [Rewards.ts](src/game-meta/rewards/Rewards.ts)。

## 目录结构

```
src/
  game-core/            # 纯 TS 战斗内核（无引擎依赖，可单测）
    board/              # 坐标、方向、地形、布局、GridBoard
    unit/               # 单位定义与运行时实例
    state/              # BattleState、BattleEvent
    pattern/            # 命中形状 + 旋转算法
    skill/              # 技能定义、命中解析、伤害/状态/地形结算
    displacement/       # 位移：push/pull/gather/swap/lineup/knockback
    pathfinding/        # 移动范围 BFS + A* 寻路
    turn/               # 回合管理、状态结算（燃烧/眩晕/冷却）
    simulator/          # BattleSimulator（纯函数）+ 预览系统
    evaluator/          # 敌方评分器（扎堆/直线/危险地形惩罚）
    ai/                 # 敌人 AI（复用模拟器枚举+评分）
    content/            # 内容注册表、关卡加载
  game-data/            # 数据驱动配置（JSON）：patterns/skills/units/levels/items + 奖励/剧情数据
  game-meta/            # 跨战斗纯逻辑：道具/背包、装配出战、奖励、存档、剧情图
  interaction/          # 引擎无关的玩家操作状态机（BattleSession → ViewModel/SessionHost）
  campaign/             # 流程编排（CampaignDirector → 标题/过场/战斗/结算/整备/结局）
  platform/
    pixi/               # 唯一表现层：PixiJS 2D 渲染、DOM HUD、输入控制器、本地存档
    wechat/             # 微信小游戏移植说明（占位）
tests/                  # vitest 测试（含贪心 bot 全战役通关的数值验证）
```

## 设计要点

- **纯函数模拟器**：`simulate(state, action) → { nextState, events }` 不修改输入。
  预览 = 在克隆态上跑同一套模拟（`previewSkill`），所以「所见即所得」。
- **事件驱动表现**：结算只产出 `BattleEvent`，渲染层据此画伤害数字/箭头/动画。
- **数据驱动**：技能/形状/单位/关卡/道具全部 JSON 配置，新增内容无需改逻辑。
- **速度初动（CT）行动顺序**：每个单位按 `speed` 充能，达阈值即行动、行动后扣除阈值，速度越高出手越频繁（见 `game-core/turn/turn.ts`）。
- **AI 复用内核**：敌人 AI 用同一个 `BattleSimulator` 枚举行动、用 `BattleEvaluator` 打分，
  内置「扎堆惩罚 / 直线惩罚 / 危险地形惩罚」，避免敌人主动送 AOE。
- **数值可验证**：`tests/campaignBalance.test.ts` 用贪心 bot 按剧情顺序实跑全部 10 关，保证战役可通关。
