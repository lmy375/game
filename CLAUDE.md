# CLAUDE.md

游戏名：**阵棋**（回合制战棋）。

## 开发标准

从现在开始，要按**生产级别**去做游戏，而不是 Demo 级别。

具体地：
- 功能做完整、健壮：覆盖边界、异常与失败路径，而不是"能跑通主流程就行"。
- 代码、命名、结构、测试都按可长期维护、可上线的标准；改动配套测试，`vitest` 全绿再算完成。
- UI、文案、手感按正式产品打磨；不留 `Demo` / `MVP` / 占位文案 / 临时 hack。
- 数值与关卡设计要经过验证（模拟或实机跑通），不靠"大概齐"。

## 坐标与格子交互

- 任何棋盘格子问题都优先检查「屏幕坐标 → Pixi stage → world → Grid.pixelToCell」这一整条链路，确保鼠标悬停、高亮、点击生效、菜单锚点使用同一套变换。
- Pixi 画布存在 CSS 缩放时，X/Y 比例要分开换算；不要假设画布显示尺寸和渲染尺寸等比。
- 任何改变棋盘视觉位置的效果（例如 world 震动、缩放、位移、相机偏移）必须基于原始布局叠加，并在结束后还原；命中测试也必须读取当前实际 world position/scale。
- 修格子错位时，不只看点击逻辑，也要同时验证 hover 高亮、技能预览、确认释放、DOM 菜单定位和触摸输入是否仍对齐。
